开了个新坑，增加了自己的跨平台批处理构建工具loon-build。项目位于[Loon-Build]这个目录下。其本质上其实就是个简化版的maven，但是针对loon项目做了很多简化，所以配置起来更加简洁。而且将加入很多约定优先的默认设置，所以仅对loon项目的打包与发布来讲，肯定比maven要简单得多（而且如导出loon代码为moon和unity这些第三方项目资源时，也肯定是自己写好的默认批处理更方便）。

目前提供了一个javase-game-sample示例，用来演示如何调用loon-build脚本，将src下的资源文件，按照build.txt中描述，直接打包为一个可单独执行的jar。稍后也会提供同类的gwt打包和android以及robovm打包示例。至于导出Unity和Mono项目的示例，则需要我写完对应0.5版的Loon-Mono和Loon-Unity才能提供……

这个小项目的理想目标，在于全部完成后，用户仅需要设定几个相关SDK所在位置，就能一键搞定所有平台的loon游戏发布。
